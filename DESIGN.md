# 初期設計メモ

・mainにすべての処理を記載する
・関数分けは後で
・エラー処理はまだ考えない
・入力は標準入力から受け付ける

# 必要な処理
・標準入力を受け付ける
・標準入力の文字列を数字と記号に分ける
・改行は消す

# Ver0.1
+の処理だけ実装
・不満点
入力の想定が＋のみ
A+Bのカタチしか計算できない
partsが2個以外だとpanicになる

# Ver0.2
2項演算以外では処理をしないように変更
ほかの演算子にも対応
・不満点
if文が単調
2項演算のみ対応
整数のみ

# Ver0.3
if文の重複を削除
unwrap()の削除
　→エラーの内容をユーザーに伝えるため
mapとcollectを使用して整数への変換
　→関数を使用せず、右辺と左辺を整数に変換するため
・不満点
整数のみ
2項演算のみ対応
3- -1などの負の整数に対応していない

# Ver0.4
浮動小数に対応
変換部分にtrim()を嚙ませることで間にある空白に対応
0で割った場合に演算を行わないように修正
・不満点
2項演算のみ対応
3 - -1などの負の計算ができない
